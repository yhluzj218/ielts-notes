## Core Philosophy

**Three keys, in order: Method → English → Practice.**

Most IELTS failures are method failures, not English failures. Fix the approach first. Only address English once the method is solid. Only increase practice volume once both method and English are in place.

**Simon's English key = vocabulary, not grammar.** Students who focus on grammar (e.g. "complex structures") are misdirecting their energy. Widening the range of words and phrases used correctly is the higher-leverage English improvement.

> ⚠️ Connie-specific note: Matt and Joe have both independently confirmed that vocabulary is already Connie's strength and is NOT the bottleneck. Simon's general recommendation to prioritize vocabulary does not override this personal assessment. See `knowledge/conflicts.md` for the recorded conflict.

## Coaching Principles

- **Diagnose before teaching** — identify the real bottleneck; don't assume vocabulary or grammar
- **Prioritise high-impact mistakes** — fix errors affecting the most marks first
- **Train by component** — decompose weak skill into sub-skills; train each separately
- **Plan before executing** — planning is non-negotiable in essays and in sprint design
- **Measurable progress** — every session must produce a measurable outcome
- **Automatic performance** — target is correct performance under timed pressure
- **Learn from others' mistakes** — analysing errors made by other students is a valid and efficient study technique; patterns in others' writing often expose blindspots in your own
- **Implicit correction awareness** — when a teacher repeats what you said using different words, that is a correction. Most students miss it. Listen for the difference between what you said and what the teacher said; that gap is the error to log.

## Anti-Patterns

| Anti-Pattern | Why It Fails |
|---|---|
| Switching methods mid-sprint | Destroys the feedback signal; causes score fluctuation — diagnosed as method inconsistency, not English decline |
| Changing method when stuck on same score | Wrong diagnosis: if stuck, the fix is English or practice, NOT a new method |
| Grammar books (systematic) | Does not build automatic performance |
| Endless mock tests without error classification | Scores plateau without insight |
| Improving everything at once | Dilutes effort; no measurable gain in any area |
| Advanced vocabulary before core method | Wrong priority order |
| Optimising language before method is solid | Fixing word choice in an off-topic essay does nothing |
| Writing body paragraphs without sentence jobs | Leads to repetition and underdevelopment |
| Repeating the same explanation in different words | Paragraph moves nowhere |
| Adding many undeveloped ideas | Breadth without depth scores low |
| Choosing structure from keywords only | Must compare 2–3 options before committing |
| Full essay before testing skeleton | Time wasted if structure is flawed |
| Reverting to old habits when writing | Long introductions, unbalanced paragraphs, conclusions with new information — method is not yet automatic |
| Learning "impressive" linking phrases | Wrong aim — does not improve score; focus on ideas and natural expression instead |

## Grammar — Key Rules from Error Analysis

> These rules emerged from Simon's student error analysis. Add to error database only when confirmed in Connie's own writing.

| Error pattern | Correction | Rule |
|---|---|---|
| Whenever... I will | Whenever... I (present) | Zero conditional: whenever/if + present tense → result in present tense. Never use "will" in the condition clause. |
| feel + -ing (relaxing) | feel + -ed (relaxed) | Adjectives describing how a person feels = -ed form. -ing = describes what causes the feeling. |
| phrasal verb used as adjective ("feel calm down") | adjective only ("feel calm") | Check part of speech: "calm down" = phrasal verb (action); "calm" = adjective (state). Don't mix them. |

## Writing — Task 2

### Paragraph Building

Every sentence in a body paragraph must have a distinct job. No two sentences should do the same job.

**Five-sentence paragraph (default structure):**

| Position | Job |
|---|---|
| 1 | Topic sentence / central idea |
| 2 | Explanation |
| 3 | Support — reason / example |
| 4 | Result / consequence / development |
| 5 | Final sentence — completes or extends the argument |

**Sentence jobs (full list):** introduce · explain · reason · example · result · predict · specific detail · complete

**No-repetition rule:** Opinion → explanation → same explanation again = bad. Opinion → explanation → example → result → development = good.

---

### Idea Development

One idea fully developed > several ideas superficially stated.

Do not: move to a new idea before current idea is complete · repeat the same point · write only general statements.

Do: explain the same idea in more depth · add specificity · support with reason / example / result / consequence / detail.

---

### Collocation Awareness (Lexical Resource)

The Band descriptor "awareness of style and collocation" means knowing which synonyms work in a given context — not just knowing synonyms exist.

**Key principle:** Not all synonyms of a word are interchangeable. The correct synonym depends on what it collocates with naturally.

Example: "The lesson was interesting."
- ✅ fascinating / engaging / absorbing / thought-provoking — all collocate with "lesson"
- ❌ attractive / entrancing / appealing — do not collocate naturally with "lesson"

**"The lesson was attractive" = wrong**, even though "attractive" is a listed synonym of "interesting."

**What this means for Connie:** When writing or paraphrasing, checking a dictionary for synonyms is not enough. Ask: does this word collocate naturally with the noun/verb next to it? A native speaker would not say it if it doesn't — and an examiner will notice.

---

### Essay Skeleton

Build the skeleton before writing the full essay.

**Four components only:**
1. Introduction — **2 sentences** (paraphrase of question + state position)
2. Topic sentence — Body Paragraph 1
3. Topic sentence — Body Paragraph 2
4. Conclusion — **1 sentence** (restate position only — no new information)

**Why it matters:** Tests whether the structure actually answers the question before writing time is spent. Prevents off-topic body paragraphs.

**Drill:** Skeleton-only practice (no full essay) as a standalone exercise.

---

### Structure Selection

Do not default to a fixed template. Choose the structure that best answers the exact question.

**Selection process:** Before writing, generate 2–3 possible structures. Choose the one that:
- directly answers the question focus
- supports a clear, consistent position
- avoids repetition across body paragraphs
- allows balanced paragraph development
- is easy for a reader to follow

---

### Practical Formatting (Task 2)

- **Word count check**: ~10 words per line → write 25+ lines to exceed 250 words
- **Paragraph separation**: leave a full blank line between paragraphs — score drops immediately if paragraphs are not clearly separated

---

### The 75% = Ideas

Grammar = 25% of score. The other 75% (Task Response + Coherence & Cohesion + Lexical Resource) can be summed up as **ideas**.

| Criterion | What it really tests |
|---|---|
| Task Response | Having good ideas that answer the question |
| Coherence & Cohesion | Presenting ideas clearly and logically |
| Lexical Resource | Expressing ideas with the right words |

**Implication:** Improve the quality of your ideas first. Grammar matters, but it is not the primary Band lever.

## Speaking — Specific Methods

### Part 1

Answer the question directly, then give one reason. One sentence is enough.

> Do not over-explain. Do not add extra details. Short, direct, complete.

Alignment: ✅ Consistent with Joe's instruction ("short, natural answers in Part 1").

---

### Part 2 — Content Generation

The most common failure: vague, general answers that don't go anywhere.

**Techniques to generate interesting content:**
1. Choose an extreme or unexpected example (not the obvious answer — a helicopter, not a car)
2. Use **named examples** — specific names, not generic categories (e.g. "tigers and rhinos", not "animals")
3. Use personal examples / anecdotes
4. Describe in specific detail

**Goal:** say something that isn't general, vague, or bland. An interesting answer forces you to use your imagination and naturally fills time.

> ⚠️ Note: "Being interesting" is not a band descriptor criterion — it's a technique. The underlying benefit is that specific, detailed answers are easier to extend to 2 minutes.

**Alignment:** Directly addresses Connie's Part 2 stamina weakness (must fill 2 min consistently — `connie_profile.md` §8.2).

---

### Cross-Skill: Think in Paragraphs

Paragraph structure applies to both Writing and Speaking.

**In Speaking:** start an idea → explain it in detail → short natural pause before the next point. This pause = the equivalent of starting a new paragraph.

**Underlying principle:** Clear, coherent thinking is the source of clear, coherent speaking and writing. If you think in paragraphs, your output will be structured without effort.

**Alignment:** Supports Joe's Part 3 principle ("extend to society-level scope") — using one idea, fully explained, is better than several ideas stated and abandoned.

## Listening — Specific Methods

### Keyword / Paraphrase Strategy

The same keyword and paraphrase approach used in Reading applies to Listening.

**Core process for MCQ and similar question types:**
1. Read the options before the audio — identify the keywords in each option
2. During the audio, listen for paraphrases of those keywords (not exact matches)
3. Verify ALL conditions in the correct option are satisfied — not just one keyword

**Critical distinction: paraphrase ≠ synonym only.**

Paraphrases can be conceptual equivalents:
- "a poor use of resources" = "a waste of funds" ← word-level synonym
- "only a minority would use it" = "it is elitist" ← conceptual equivalent (elitist = reserved for a small group)

**Why this matters for Connie:** This directly explains the mechanism behind "Premature Conclusion Before Verification" in Listening. Hearing one keyword from an option is not enough — the full condition set must be mapped. An option that shares a keyword with the transcript but does not satisfy all conditions is a distractor.

**MCQ verification rule:** For a multiple-choice answer to be correct, ALL stated conditions in that option must have paraphrase equivalents in the audio. One match is not sufficient.

### Disprove Wrong Answers — Active Elimination

For every MCQ, the full process is two-directional:

1. Find keyword evidence **for** the correct answer
2. Find keyword evidence **against** the wrong answers

Wrong answers must be actively disproved — not just ignored.

Examples of disproof keywords:
- Option says "in different countries" → transcript says "concentrate on this country" → eliminated
- Option says "funding" → transcript says "best if we don't go into funding" → eliminated
- Option says "difficult to read / long to read" → transcript does not mention either → eliminated (Not Given)

**Why this matters for Connie:** Premature Conclusion Before Verification is caused by stopping at "found evidence for one option." The full method requires also confirming the other options are eliminated by the transcript.

### Speaker Agreement / Stance Detection

Speakers do not always say "I agree." Agreement is signalled through response markers.

Pattern: Speaker A makes a claim → Speaker B responds positively → both are aligned.

Example: Stewart says "you won't find up-to-date information." Trudie replies: "That's an important point." = They agree.

Common agreement signals: "That's a good point" · "Exactly" · "I think so too" · "That's true" · "Yes, definitely" · "I agree" · "Absolutely"

Common disagreement signals: "But..." · "Actually..." · "I'm not sure about that" · "I don't think so"

### Paraphrase Types — Extended

Paraphrases are not only word-level synonyms. They can be:

| Type | Example |
|---|---|
| Synonym | "a poor use of resources" = "a waste of funds" |
| Conceptual equivalent | "only a minority" = "it is elitist" |
| Example used as illustration | "changes in society" = "external developments, like the fact that more people can read" |
| Logical inference | "copyright lasts 70 years after death" → "they are generally old" |

The hardest paraphrases are the last two types. These require the listener to bridge an inference gap, not just match a synonym.

### MCQ Method (Step-by-Step)

**Before the recording:**
1. Read questions and all choices — not just the question stem
2. Underline only the **differentiating keywords** in each choice: words that create a difference in meaning *between* options
3. Do not underline words that are shared across all options — those will not discriminate the answer

**Differentiating keywords — the key concept:**
MCQ options often share many words. The words that matter are those where A says one thing and B says another. Those are the only words worth tracking in the audio.

Example: If options A/B/C all mention "students" but differ on "cost / items / time" — underline only cost / items / time.

**During the recording:**
- Only look at the underlined words — do not re-read full sentences
- Track which underlined word gets a paraphrase equivalent in the audio
- Expect a trick: more than one option may be mentioned; one will be eliminated by a qualifier

**Between sections (transition time):**
- Do NOT check previous section answers
- Use every break to read ahead and underline keywords for the next section
- Time spent confirming what you already wrote is wasted time

**If unsure:** Make a best guess and move on immediately. Never leave blank.

---

### Three Listening Trick Types

IELTS Listening deliberately includes traps. Three types appear repeatedly:

**Trick Type 1 — Multiple options mentioned, one eliminated by a qualifier**

The speaker mentions more than one number/option, but only one satisfies the condition in the question.

Process: The question specifies the condition → track which answer is tied to that condition in the audio.

Example:
- Question: "Number of items allowed (members of public)"
- Audio: "12 items for students... However, it's only 8 for members of the public."
- Answer: 8 (the qualifier "members of the public" determines which number is correct)

**Trick Type 2 — Speaker changes their answer (self-correction)**

The speaker gives an initial answer, then explicitly corrects it. The correct answer is the correction, not the first answer.

Self-correction signals: "Oh no, sorry..." · "Actually, it's..." · "I mean..." · "Sorry, I made a mistake..."

Example:
- Audio: "Yes, he's here on Thursday evenings. Oh no sorry, Friday."
- Answer: Friday (not Thursday)

**Trick Type 3 — Conditional branches (different answers for different people/situations)**

The speaker describes two or more scenarios with different answers. The question specifies which scenario applies.

Example: "£125 per year or £25 if you've got a current student card."
- Question says "without current student card" → answer is 125, not 25.

---

**Rule for all three trick types:** Do not write the first answer you hear. Keep listening until you confirm which answer satisfies the exact condition stated in the question.

**Connection to Connie:** All three tricks exploit "Premature Conclusion Before Verification." The trap is always the same: a plausible answer is mentioned first, and the correct answer comes after a qualifier, correction, or condition. Keep listening past the first number/word.

---

### Transcript Analysis — Study Method

After every Cambridge Listening practice:

1. Check answers
2. Read the transcript for every question answered incorrectly **and** every question answered correctly but uncertainly
3. For each question: identify the exact words in the transcript that correspond to the correct answer
4. Identify the words that **eliminate** each wrong option
5. Note whether the paraphrase was: synonym / conceptual / example / inference

**Purpose:** Builds automatic paraphrase recognition. Moves from reactive listening (hoping to catch it) to trained pattern recognition.

**Step 6 — Classify the cause of each mistake:**

| Cause | What it means |
|---|---|
| Word / phrase | Failed to recognise a paraphrase or specific word |
| Misunderstood | Understood words but not the meaning or function |
| Stuck on previous question | Missed the current answer because still processing the last one |
| Speed / volume | Could not process the amount of information quickly enough |

Classifying the cause is required — checking the answer without diagnosing WHY it was wrong produces no improvement.

## Diagnostic Framework — Score Fluctuation

If scores go up and down between attempts:

**Diagnosis:** The method is inconsistent — not that English has declined.

**Fix:** Do not change the method. Identify which step of the method was missed or changed.

**Rule:** Progress is only measurable when the same method is used every time. Fluctuation without method change = evidence of English ceiling. Fluctuation with method change = method inconsistency.

---

## Diagnostic Framework — Stuck at Band 6.5

**Band 6.5 = already Band 7 in 50% of the criteria.** The method is working.

Example: TR 7 + CC 7 + LR 6 + GRA 6 = 6.5 average. The student is not failing — they need to improve just ONE weak criterion to shift the average to 7.

**Steps:**
1. Identify which criterion is dragging down the average
2. Improve only that one area
3. Do NOT change the method — it is already producing Band 7 in some criteria

**Common mistake:** Students stuck at 6.5 think they need a new technique or book. They don't. They need a small, targeted improvement in the right place.

---

## Diagnostic Framework — Time Pressure as Weakness Detector

**Technique:** Occasionally do tasks faster than the allowed time (e.g. Task 2 in 30 min, Reading in 50 min) to expose weak points.

**Purpose:** Some weaknesses only appear under pressure. Going faster surfaces them.

**Important:** This is a diagnostic tool only — use occasionally. Most practice should be slow and careful. Do not use this as a regular training mode.
