## Core Argument

Most IELTS students fail not because of weak English, but because they are using the wrong method.

Fixing the method first produces faster improvement than studying more English.

Priority order:

1. Method — know exactly what to do in each task
2. English — only if method is solid and English is the bottleneck
3. Practice volume — meaningful only after method and English are in place

---

## Nine Core Beliefs

1. Method before English before practice — fix the approach first, not the language.
2. Master one method and stick to it — switching methods destroys feedback signals.
3. Small improvements compound — fixing one specific error is worth more than broad practice.
4. Fix one bottleneck at a time — only one primary weakness per sprint.
5. Deliberate practice beats repetition — targeted error correction over more mock tests.
6. Build skill, not knowledge — knowing a rule is not the same as applying it under pressure.
7. Grammar is not the centre of IELTS — grammar matters but is not the primary Band lever.
8. Authentic English exposure helps after Band 7+ — immersion only works once core skills are built.
9. Automatic performance is the goal — the skill must work under timed exam conditions.

---

## Coaching Principles

- Diagnose before teaching — identify the real bottleneck; do not assume it is vocabulary or grammar.
- Prioritise high-impact mistakes — fix errors that affect the most marks first.
- Train by component — decompose a weak skill into sub-skills; train each separately.
- Plan before executing — in both essay writing and sprint design, planning is non-negotiable.
- Measurable progress — every session should produce a measurable outcome.
- Automatic performance — the target is performing correctly under timed pressure.
- Give specific answers — for every IELTS task, there is a correct method.

---

## Sprint Design Rules

1. Never change methods during a sprint — pick one, commit, measure.
2. One primary weakness per sprint (maximum).
3. One secondary weakness at most — never two high-priority items simultaneously.
4. Decompose weakness to sub-skills before designing practice.
5. Training (targeted exercises) over testing (mock tests) when below target Band.
6. Plan the sprint before executing — daily targets must be defined in advance.
7. Every daily session should be measurable (completion rate, error count, time taken).
8. Build in reflection: what worked, what did not.
9. Repeated production practice for Writing and Speaking (not just review).
10. Use authentic English sources (BBC, Guardian, native podcasts) — not IELTS social media content.

---

## Anti-Patterns

- Switching methods mid-sprint — destroys the feedback signal.
- Using grammar books systematically — does not build automatic performance.
- Doing endless mock tests without error classification — scores plateau without insight.
- Improving everything at once — dilutes effort, no measurable improvement in any one area.
- Learning advanced vocabulary before mastering core task method — wrong priority order.
- Optimising language before method is solid — fixing word choice in an off-topic essay changes nothing.
