## Consensus Principles

### 1. Task Response Before Language

| Field | Value |
|---|---|
| Principle | Identify the correct question type and respond to it fully before worrying about grammar or vocabulary |
| Agreement | ✅ Simon + Matt + Joe |
| Simon | Method-first philosophy; wrong question type = automatic band cap |
| Matt | "切題" is always the first correction — confirmed across all 6 writing sessions |
| Joe | 2026-07-02 (writing lesson): reframed Connie's own "write down my ideas" answer as "clearly communicate your opinion and the reasons for your opinion" — same TR-first frame |
| Sprint implication | **Task Response must always be Sprint Priority #1 for Writing** |

---

### 2. Plan 5 Minutes Before Writing

| Field | Value |
|---|---|
| Principle | Allocate planning time before starting the draft; write without editing during drafting |
| Agreement | ✅ Simon + Matt |
| Simon | Sprint Design Rule #6: Plan before execute; Core Belief #1 |
| Matt | Consistent instruction across all lessons: plan structure, then write |
| Sprint implication | Include timed planning as a required component of every writing practice session |

---

### 3. Idea Development: Point → Explain → Example → Result

| Field | Value |
|---|---|
| Principle | Never stop at a topic sentence; develop each point with explanation, example, and consequence |
| Agreement | ✅ Matt (explicit) + Simon (explicit — Five-Sentence Paragraph Method) + Joe |
| Matt | Explicit framework taught 2026-06-19; repeated in feedback across sessions |
| Simon | Five-sentence paragraph method; sentence jobs rule; no-repetition rule — `10_knowledge_base/coaches/simon/distilled.md` |
| Joe | 2026-07-02 (writing lesson): taught identical skeleton (Intro → Body 1 = Reason 1 + Main Idea/Explain/Example → Body 2 = Reason 2, same structure → Conclusion); explicitly said low scores come from "reasons not explained" and mixing ideas in one paragraph |
| Sprint implication | Body paragraph development is Priority #2 for Writing; verify chain in every draft |

---

### 3a. Every Sentence Must Have a Distinct Job

| Field | Value |
|---|---|
| Principle | In a body paragraph, no two sentences should do the same job; each sentence must move the paragraph forward |
| Agreement | ✅ Simon (explicit) — Matt (implicit: same chain logic) |
| Simon | Sentence Jobs Rule: introduce · explain · reason · example · result · detail · complete |
| Matt | Implicit — same requirement produced by Point → Explain → Example → Result chain |
| Sprint drill | After writing a body paragraph, label the job of every sentence; remove any that repeat an earlier job |

---

### 3b. Plan the Essay Skeleton Before Writing

| Field | Value |
|---|---|
| Principle | Before drafting, write only four things: intro position + topic sentence 1 + topic sentence 2 + conclusion position |
| Agreement | ✅ Simon (explicit) — Matt (aligned: plan first rule) |
| Simon | Essay skeleton practice as a standalone drill; tests coherence before any writing time is spent |
| Matt | Consistent: 5-minute planning, structure first |
| Sprint drill | Skeleton-only drills (no full essay) to build structure speed |

---

### 3c. Compare 2–3 Structures Before Committing

| Field | Value |
|---|---|
| Principle | Do not default to a fixed template; select the structure that directly answers the question focus |
| Agreement | ✅ Simon (explicit) |
| Simon | Structure Selection Rule — see `10_knowledge_base/coaches/simon/distilled.md` |
| Matt | Not explicitly stated; implied by question-type identification priority |

### 4. Grammar From Own Errors, Not Systematic Study

| Field | Value |
|---|---|
| Principle | Fix grammar errors by tracking what appears in Connie's own writing — not from a textbook |
| Agreement | ✅ Simon + Matt |
| Simon | Anti-pattern: grammar books; build skill from production, not study |
| Matt | Corrects specific errors from Connie's essays — no systematic grammar curriculum |
| Sprint implication | Use `09_coach/errors/grammar_db.md` as the sole grammar reference — not grammar guides |

---

### 5. Vocabulary Is Not the Priority

| Field | Value |
|---|---|
| Principle | Do not allocate sprint time to vocabulary acquisition for Writing |
| Agreement | ✅ Simon + Matt + Joe |
| Matt | Explicit: "instinctively, not a vocabulary problem"; LR consistently Band 7 |
| Joe | Confirmed: vocabulary is a strength (speaking assessment, 6/01). **Writing-specific mechanism added 2026-07-02**: forcing higher-register words produces wrong collocation/register/sense and actively lowers score — see `10_knowledge_base/coaches/joe/distilled.md` § Writing — Mistake #1 |
| Simon | Anti-pattern: advanced vocab before method |
| Sprint implication | **Never suggest vocabulary work as a writing improvement strategy.** When reviewing drafts, flag any word choice that looks reached-for rather than natural — this is now a concrete editing check, not just a general rule. |

---

### 6. Finished Essay Beats Unfinished Perfect Essay

| Field | Value |
|---|---|
| Principle | Complete the essay under time pressure; do not sacrifice completion for perfection |
| Agreement | ✅ Simon + Matt + Joe (Joe added 2026-07-15 via overthinking mechanism) |
| Matt | Repeated: "Finished > Perfect"; Connie identified as over-editing during drafting. **Mechanism specified 2026-07-15**: write quickly without re-reading from the start, leave 5 minutes at the end for a grammar pass; "write how you would speak — thinking too much makes it worse" (her spoken grammar is mostly correct, so drafting like speech is safe for her specifically) |
| Simon | Core Belief: automatic performance under exam conditions is the goal |
| Joe | Same overthinking diagnosis from the speaking side (hesitation is the bottleneck, not language); writing lesson 7/09: don't chase perfect examples — general ones are fine |
| Sprint implication | Every timed writing practice must finish within the time limit — partial essays are not valid. Timed drills should enforce: no mid-draft re-reading, 5-minute end check reserved for G007/articles/spelling |
