# Skeleton — Problem + Solution

> Last updated: 2026-07-01

---

## 辨識規則

**只看最後一句指令：**

| 指令關鍵字 | 判斷 |
|---|---|
| `What are the reasons / causes?` + `What solutions / measures?` | ✅ Problem + Solution |
| `Why is this happening?` + `How can this be solved?` | ✅ Problem + Solution |
| `What are the problems?` + `What can be done?` | ✅ Problem + Solution |

**注意：** Intro 必須同時預告 reasons 和 solutions 兩件事，不能只說「I will discuss this issue.」

---

## Variation A — Paired（因果配對）

每個 body 處理一組「原因 → 對應解決方案」。因果直接連結，讀者不會搞混哪個解法對哪個原因。

```
Intro:   Paraphrase 問題 + 預告你會給 reasons 和 solutions
Body 1:  Cause 1 → Solution 1（先說原因，再說解法）
Body 2:  Cause 2 → Solution 2
Concl:   Restate 整體觀點
```

**Intro 模板：**
> `[Paraphrase 問題].` `This essay will explore the main reasons behind this trend and suggest several measures to address it.`

**何時用：** 每個原因有自己的專屬解法，配對關係清楚。

---

## Variation B — Separated（原因獨立、解法獨立）

Body 1 討論所有原因，Body 2 討論所有解法。

```
Intro:   Paraphrase 問題 + 預告
Body 1:  Cause 1 + Cause 2
Body 2:  Solution 1 + Solution 2
Concl:   Restate
```

**何時用：** 解法不是針對某一個原因，而是同時解決兩個原因。

**⚠️ 注意：** 如果 Solution B 只能解決 Cause 2，用 Variation A 更清楚，不要用這個結構。

---

## Intro 常見錯誤

❌ 不能寫：
> "I will show some reasons and solutions in the following paragraph."

✅ 應該寫（點名兩件事都會處理）：
> "This essay will explore the main reasons behind this trend and suggest several measures to address it."

## Skeleton 自我檢查

- [ ] Intro 有同時預告 reasons 和 solutions？
- [ ] 選了 Paired 還是 Separated？選的理由是什麼？
- [ ] Paired 的話：每個 body 的原因和解法真的有直接因果關係？
- [ ] Separated 的話：Body 2 的解法真的可以同時解決兩個原因？
- [ ] Conclusion 只重申整體觀點，沒有加新論點？
