## 口說原則

- 別過度思考（老師反覆強調 don't overthink）。
- Part 1：直接回答、答完就停、只答被問的問題；長短看題目決定。
- Part 2：
  - 一定要講滿 2 分鐘（提早停比講太多更糟）。
  - 用題目本身搭結構（開頭=這是什麼 / 中段=細節 / 結尾感受）。
  - 用 linking phrase 串接，但「不需要連成完整故事、不需要結論」。
  - 準備時間（1 分鐘）也要用來「篩選好不好講的題材」，
    避開詞彙門檻高的主題。
- Part 3：講「一般人 / 社會普遍現象」，不要講自己（這是 Part 1/2 才做的）。
- 高分關鍵 = 傳達大量自然的資訊與細節，不是用複雜語言。
  簡單語言反而溝通更好。

### 不要做的事（2026-06-01 起，Matt 明確強調）
- 不要跟讀/仿效母語者：你的英文已夠好，複製反而干擾自然溝通
  （「It's got to be your communication during the exam, not imitating a native speaker.」）
- 不要看社群媒體/YouTube 的 IELTS 備考內容（考官不想看到那些「技巧」）
- 不要背稿/背答案：考官故意問意外問題，看你怎麼應對；答案太像準備好的會扣分

### 連讀 / 自然語調（試聽課點出）
- 「You're pronouncing words too individually / too correctly.」
- 例：`I would like to go` → `I'd like to go`（母語者不把每個字都發清楚）
- 這比個別單字發音更影響分數，也影響聽力理解

### Shadowing（跟讀）
- 有幫助，但要先上完「句子語調課（intonation lesson）」再做
- 先了解方法，跟讀才有效；學會前不要盲目跟讀

### 亞洲 vs 歐洲考生平衡
- 亞洲：大量備題/背答案 → 聽起來太機器人、不自然
- 歐洲：只靠放鬆說話 → 有時沒東西可說
- 最佳平衡：準備常見主題的想法，但不背稿；自然說話

### 備考時間策略
- 現在：重點改文法 + 發音（流暢度暫時下降是正常的，不要擔心）
- 考前：文法/發音穩定後，只需要思考題目和想法

## 寫作原則

- 低分主因是「沒答到題 + 結構不對」，不是英文不好。這些是技術問題，
  一講就能改、能短期跳分。
- 先修「切題」和「結構」，不要先談詞彙文法。
- Task 2 讀題：
  - 看清楚問法：agree/disagree ≠ positive/negative
    （agree/disagree 最常考，約佔 40–50%）。
  - 改寫題目時不能改掉原意（題目用 nobody/all 這種強度字，
    不能改寫成 less/some，否則立場就變了）。
  - 要明確表態 + 給理由，不要只列優缺點。
  - 觀點要在開頭就說，不要埋到最後。

### Task 1 信件原則（2026-06-19）
- 每個 bullet point 各開一段，不能全部擠在一起。
- 正式結尾規則：Dear Sir/Madam → Yours faithfully + 全名；Dear + 名字 → Yours sincerely。
- 投訴信語氣可稍強硬，不需要太客氣。
- 字數不要剛好停在 150 字，多寫一點才有機會拿高分。

### Task 1 非正式信件補充（2026-06-26）
- 給朋友的信（Dear + 名字），結尾不一定要 "Yours sincerely"，更自然的選項：
  - `Love, [名字]`（關係親密）
  - `Take care, [名字]`
  - `All the best, [名字]`
  - `Hope to catch up soon, [名字]`
  - `Keeping in touch, [名字]`
  - `Regards` 適合 email，正式信件太生疏，給好朋友更不合適。
- 寫給特定朋友時，想像真實的那個人：
  - 加入個人細節（為什麼你知道他在做這個報告、你們的關係）
  - 信的語氣要像真的在跟那個人說話，不是像說明書
- 不回答題目的額外資訊 = **負分**，不是中性：浪費字數 + 增加犯錯機會，直接扣 Task Achievement。

### 討論文（Discussion Essay）結構（2026-06-26）
- 題型特徵："discuss both views and give your own opinion"
- 選一個你**比較同意**的立場

**結構：**
- Body 1 = **對立方（比較弱的那邊）**
- Body 2 = **自己的立場（比較強的那邊）**
- Conclusion = 總結兩個觀點 + 重申自己的立場

**Body 1 最關鍵、也最容易犯錯的地方：**
- 大多數人的錯誤：Body 1 只是說「有些人認為 X，因為 Y」，點到即止。
- 老師說：你在 Body 1 的**工作是替對立方辯護**——即使你個人不同意，也要像你真的理解並認同那個立場一樣去寫。要給解釋、給例子，讓那個觀點聽起來有說服力。
- 老師的政治例子：就算我不認同某政治人物的移民政策立場，我也要能寫出一段話，清楚說明為什麼有人會支持那個立場——這就是 Body 1 的精神。
- 考官看得出你是否真的思考過另一邊，還是只是草草敷衍。

**進階版（老師提到但尚未教）：**
- 當你的立場不是完全站某一邊，而是「介於兩者之間」時，有另一種寫法。老師說這次先不講，之後再教。

### Task 2 "to what extent agree/disagree" 補充（2026-06-26）
- 題目有 "because + 前提" 時，**必須直接回應那個前提**，不只是說自己的理由
- 承認對方前提的部分合理性，再說為什麼你的理由更重要（讓步 + 反駁）
- 例：「語言少 = 生活更方便」→ 先承認「確實便利」，再說「但文化損失遠大於便利」
- 只說「語言很重要」不回應「便利」這個點，Task Achievement 拿不到高分

### Task 2 原則（2026-06-19）
- although 用法：放句首造對比（Although I like pizza, I try not to eat it）；句內不再加 however 或 but，兩者只能選一個。
- 段落展開結構：點出觀點 → 解釋為什麼 → 給例子 → 說明例子跟題目的關係（不能用 example 結束段落）。
- 題目有 "because + 前提" 子句 → 要直接回應那個前提，不只是說「後果不好」。
- `due to` ≠ `because`；要接子句時用 `because` 或 `due to the fact that`。
- 說了幾個 reasons，body paragraph 就寫幾段，數量要一致。
- 考場只有 5 分鐘想題：簡單想法說清楚 > 複雜想法說不清楚。

### 我的寫作普遍弱點（2026-06-26 老師總結）
- **段落資訊量不均衡**：有些段落資訊太少（甚至只有一句），有些段落資訊太多太密。老師說這是目前最明顯的結構問題。
- 理想狀態：每個段落的份量大致相當，每段都有足夠的展開（觀點 + 解釋 + 例子）。
- 修正方式：如果某段寫到很長，先問自己「這段有沒有內容其實是回答另一個 bullet 的？」——有的話就移過去，不要硬塞在同一段。
- 老師原話（大意）："My main advice is to aim for a clearer structure, where each paragraph is related to each bullet point, and each paragraph has roughly equal weight."

### 練習建議（2026-06-19）
- 練寫作至少要保留 1 小時，15 分鐘不夠，省下來做口說和聽力。
