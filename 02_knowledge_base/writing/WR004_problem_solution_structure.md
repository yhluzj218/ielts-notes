# WR004 — Problem + Solution Essay Structure

## Core Rule

The introduction **must signal BOTH** the reasons AND the solutions. If it only signals one, Task Response is incomplete.

> ❌ "I will show some reasons and solutions in the following paragraph." (too vague)
> ✅ "This essay will explain the main reasons behind this trend and suggest several measures to address it."

## Recommended Structure (Paired)

```
Introduction
  → Paraphrase the problem
  → Signal BOTH: reasons AND solutions

Body 1 — Reason 1 + Solution 1
  → State the cause
  → Explain why it's a cause
  → Immediately follow with the corresponding solution

Body 2 — Reason 2 + Solution 2
  → State the second cause
  → Immediately follow with the corresponding solution

Conclusion
  → Restate the main point
  → Optional: compare solutions (e.g., which is more effective long-term)
```

**Why paired over separate:** Each cause is immediately linked to its solution. Easier to follow. Avoids "which solution solves which reason?" confusion.

## Alternative Structure (when solutions address all causes together)

```
Introduction
Body 1: Reasons (2 causes)
Body 2: Solutions (2 solutions)
Conclusion
```

Use ONLY when both solutions genuinely address both causes simultaneously.

## Introduction Template

> `[Paraphrase problem]. This essay will explain the main reasons behind this trend and suggest several measures to address it.`

**Example:**
> "Excessive clothing consumption has become a significant issue in modern societies. This essay will explain the key causes of this behaviour and propose effective ways to reduce it."

## Conclusion tip

Conclusion can compare solutions by timeframe — this shows mature thinking:
> "While taxation may reduce consumption in the short term, education is likely to be more effective in the long run."

## Introduction self-check (Problem+Solution only)

After writing your introduction, verify:
- [ ] Does it name the **reasons** component?
- [ ] Does it name the **solutions** component?
- [ ] Is it specific enough? (not just "I will discuss...")

## Common mistakes

❌ Introduction only mentions reasons, not solutions
❌ Introduction uses vague language: "I will show some things..."
❌ Body 1 = all reasons, Body 2 = all solutions — but solutions don't pair clearly with reasons
❌ Conclusion adds new arguments
