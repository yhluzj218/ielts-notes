# WR005 — Skeleton Principle + Body Paragraph Development Chain

## Part 1: The Skeleton

### What a skeleton IS

A skeleton decides the **structure** of the essay — not the content.

```
Introduction: paraphrase + position / task statement
Body 1:       topic sentence (ONE clear main idea)
Body 2:       topic sentence (ONE clear main idea)
Conclusion:   restate position (one sentence)
```

### What a skeleton is NOT

- Not a draft — do not write full sentences in the skeleton
- Not a content plan — no examples, reasons, or details
- Not a vocabulary list

### Skeleton self-check

- [ ] Does the introduction signal what the essay will cover?
  - Problem+Solution: does it name BOTH reasons AND solutions?
- [ ] Does Body 1 have only ONE main idea?
- [ ] Does Body 2 have a DIFFERENT main idea from Body 1?
- [ ] Does the conclusion restate (not add new points)?
  - Comparing short-term vs long-term in conclusion is valid.

---

## Part 2: Body Paragraph Development Chain

### Required chain (Matt's rule)

Every body paragraph must develop through these stages:

```
[Point]    → State the main idea (topic sentence)
[Explain]  → Why / how does this work?
[Support]  → Reason / example / specific detail
[Result]   → What is the consequence?
[Complete] → Close or extend the argument (optional)
```

**Critical rule:** Never end a paragraph with an example. The example must connect back to the argument.

### Self-check after writing each paragraph

- Does every sentence add something new?
- Are two sentences doing the same job? → Remove the weaker one
- Did you reach [Result] level?
- Is the paragraph roughly equal in weight to the other body paragraph?

### Common mistakes

❌ `Point → Example → Stop` (missing Explain and Result)
❌ Body 1 is 5 sentences, Body 2 is 1 sentence (unequal weight)
❌ Two body paragraphs argue the same thing in different words

---

## Writing Process (Always Follow)

```
Read question → Identify TYPE → Choose STRUCTURE → Write SKELETON → Write ESSAY
```

**Never skip a step.** Jumping to writing without a skeleton leads to underdeveloped paragraphs.
